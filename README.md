# NailHome — лендинг для мастера маникюра на дому

Адаптивный продающий лендинг для мастера маникюра с выездом на дом. Проект сделан как пример комплексной работы: **UX‑структура → тексты → современный UI → аккуратная верстка (HTML/CSS/JS)**.

## 🧩 Задача

Создать легкий и понятный лендинг для мастера маникюра на дому, где основной сценарий — **запись через WhatsApp**, а также:

- показать **ценность формата на дому** (удобно, безопасно, экономит время),
- прозрачно донести **услуги и прайс**,
- обозначить **зону выезда**,
- усилить доверие за счет **отзывов**.

---

## 🏗 Структура лендинга

1. **Hero‑блок**  
   Сильный оффер «Салонный маникюр у вас дома за 60–90 минут», подзаголовок, список выгод и крупная кнопка записи в WhatsApp.

2. **Почему на дому — это удобно**  
   Четыре карточки преимуществ: время, гигиена, комфорт, прозрачный прайс.

3. **Услуги и прайс**  
   Сетка карточек услуг с описанием, временем и ценами. Отдельно выделен тариф‑«Хит».

4. **Зона выезда мастера**  
   Чипсы с районами, поясняющий текст и CTA‑карточка с кнопкой «Уточнить выезд».

5. **Отзывы + финальный CTA**  
   Три отзыва клиенток и заключительный блок с призывом забронировать ближайшее свободное время в WhatsApp.

## ✨ Основные фичи

- **Mobile‑first верстка** — лендинг сначала спроектирован под мобильные, затем масштабирован под планшеты и десктоп.
- **Фокус на конверсии** — повторяющиеся CTA‑кнопки WhatsApp в ключевых блоках.
- **Чистый UI:** нежная розово‑нюдовая палитра, мягкие тени, карточный layout, плавные hover‑эффекты.
- **Анимация появления блоков** при прокрутке через `IntersectionObserver`.
- **Без сторонних библиотек** — только нативный HTML/CSS/JS.

---

## 🛠 Технологии

- **HTML5** — семантическая разметка, структура из 5 основных секций.
- **CSS3**  
  - кастомные CSS‑переменные для цветовой палитры и отступов,  
  - flexbox и CSS Grid,  
  - адаптив через media‑queries,  
  - плавные переходы и тени.
- **JavaScript (Vanilla)** — анимация появления секций при скролле (`IntersectionObserver` с graceful fallback).

# Торты на заказ — лендинг «Sweet Atelier»

Одностраничный продающий лендинг для персонального кондитера: тёплая «домашняя» атмосфера, современный интерфейс и упор на индивидуальный подбор торта под запрос клиента.

## Оффер и концепция

**Оффер:** «Торт, сделанный лично для вас за 24 часа»  
**Суть:** клиент сразу понимает, что торт будет:

- **под его праздник** — учитываются повод, количество гостей, бюджет;
- **под его вкус** — выбор начинки, веса и декора;
- **без риска** — кондитер присылает эскиз и уточняет детали до приготовления.

Фокус лендинга — провести пользователя по понятному пути: эмоция → выбор типа торта → понимание цены → доверие к процессу → простая заявка.

## Структура лендинга (5 блоков)

1. **Hero / Оффер**
   - Заголовок и подзаголовок с ключевыми выгодами.
   - Краткие буллеты (вес, натуральные продукты, доставка).
   - Две кнопки: «Рассчитать торт» и «Посмотреть варианты».
   - Доверительные элементы: количество заказов и рейтинг.
2. **Каталог видов тортов**
   - Карточки: классические бисквитные, муссовые/зеркальные, тематические/фигурные.
   - Для каждой: краткий сценарий использования + вес/цена.
3. **Вес и цены**
   - Таблица: вес → количество гостей → ориентировочная стоимость.
   - Блок «Что влияет на цену» и пояснение про прозрачный расчёт.
4. **Как я работаю**
   - 4 шага от заявки до доставки, простым, «человеческим» языком.
   - Блок с гарантиями качества и подхода к работе.
5. **Форма заявки**
   - Форма с обязательными полями (имя, телефон, дата, описание торта).
   - Отдельный чекбокс «нужно срочно».
   - Микротекст: заявка ни к чему не обязывает — сначала обсуждение и расчёт.

## Технологии

- **HTML5** — семантическая разметка, якорная навигация.
- **CSS3 (mobile-first)** — адаптивная сетка, кастомные переменные, плавные hover-эффекты.
- **Vanilla JS** — бургер-меню для мобильных, плавный скролл до блоков, простая валидация формы и сообщение об успешной отправке (без бэкенда).

Без сторонних библиотек и сборщиков — лёгкий статический проект, который можно развернуть на любом хостинге (GitHub Pages, Vercel, Netlify и др.).

## Как адаптировать под свой бренд

- **Тексты:** замените название кондитера, город и контактный телефон в `index.html`.
- **Цвета:** при желании обновите палитру через CSS-переменные в начале `styles.css`.
- **Цены и веса:** отредактируйте таблицу в блоке «Сколько стоит ваш торт».
- **Форма:** при наличии бэкенда привяжите форму к своему обработчику/CRM вместо демонстрационной обработки на фронтенде.
